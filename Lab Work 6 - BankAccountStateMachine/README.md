# Lab Work: Bank Account Finite State Machine (FSM)

## Цель работы
Реализовать банковский счёт с использованием конечного автомата состояний.  
Программа позволяет открывать и закрывать счёт, вносить и снимать средства, погашать задолженность и отслеживать историю операций.  
Интерфейс отображает текущий баланс, состояние счёта и историю транзакций.

## Состояния счёта

- **Closed** — счёт закрыт (начальное и конечное состояние)  
- **Good** — положительный баланс, средства доступны для снятия  
- **Overdrawn** — отрицательный баланс, требуется погашение задолженности  

## Основные операции

- **OpenAccount** — открывает счёт с положительным начальным балансом; состояние → Good  
- **Deposit** — пополнение счёта; при переходе из Overdrawn на положительный баланс состояние → Good  
- **Withdraw** — снятие средств; при уходе баланса в минус состояние → Overdrawn  
- **RepayDebt** — погашение задолженности; состояние → Good  
- **CloseAccount** — закрытие счёта только при положительном балансе и отсутствии долга; состояние → Closed  

## Механизм работы конечного автомата

- Каждая операция проверяет текущее состояние счёта и при необходимости переводит его в новое состояние  
- Автомат обеспечивает корректность операций: нельзя снимать с закрытого счёта или закрыть счёт с долгом  
- Интерфейс автоматически обновляется при изменении состояния или баланса через события `PropertyChanged`  

## Особенности реализации

- Чёткое разделение логики состояний и операций счёта  
- Визуальная демонстрация текущего состояния и баланса  
- История операций позволяет отслеживать все изменения счёта  

## Результат работы

- Возможность безопасного управления банковским счётом через графический интерфейс  
- Автомат корректно обрабатывает переходы между состояниями при всех допустимых операциях  
- Программа предотвращает некорректные действия, например, снятие средств с закрытого счёта или закрытие при задолженности
