# Lab Work: Petri Net Simulation of Order Processing Automaton

## Цель работы
Смоделировать **автомат обработки заказов** с использованием сети Петри.  
Автомат последовательно выполняет заказы, возвращаясь в исходное состояние ожидания после завершения каждого заказа.

## Принцип работы

- **Автомат**:
  1. Находится в состоянии ожидания заказа.
  2. При поступлении заказа начинает его выполнение.
  3. Завершает выполнение заказа.
  4. Отправляет заказ на доставку.
  5. Возвращается в состояние ожидания.

- **Сеть Петри**:
  - **Позиции (места, круги)** — условия системы:
    - `a` — автомат ждёт заказа
    - `b` — заказ поступил и ждёт
    - `c` — автомат выполняет заказ
    - `d` — заказ выполнен
  - **Переходы (прямоугольники)** — события:
    1. Заказ поступил
    2. Начать выполнение заказа
    3. Закончить выполнение заказа
    4. Заказ посылается на доставку

- **Маркировка (токен)**:
  - Начальное состояние: токен в позиции `a` (автомат ждёт).
  - После полного цикла (`a → b → c → d → a`) система возвращается в начальное состояние.

## Особенности реализации

- Каждое событие проверяет возможность срабатывания перехода (`isEnabled`) и перемещает токен между позициями (`fire`).  
- Моделирование позволяет отслеживать состояние автомата на каждом этапе выполнения заказа.  
- Простая визуализация сети Петри может показывать движение токена между позициями.  

## Возможности программы

- Имитация последовательной обработки заказов.  
- Визуализация текущего состояния автомата и позиций сети Петри.  
- Поддержка циклической работы системы: автомат возвращается в ожидание после завершения заказа.  

## Использование

1. Запустить программу/открыть HTML-файл в браузере.  
2. Наблюдать за перемещением токена по сети Петри в реальном времени.  
3. Каждый цикл моделирует обработку одного заказа.
